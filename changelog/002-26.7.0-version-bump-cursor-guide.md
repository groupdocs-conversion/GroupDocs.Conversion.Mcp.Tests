# 26.7.0 — target the 26.7.0 server release + add a Cursor guide

## What changed
- Bumped the package-under-test from `26.5.2` → **26.7.0** across every pin: `Directory.Build.props` (`McpPackageVersion`), `.github/workflows/integration.yml` (`workflow_dispatch` default + resolved env default), `examples/*.json`, `how-to/*.md`, `docker-scripts/README.md`, `README.md`, `AGENTS.md`, and `llms.txt`. Historical changelog entries keep their original versions.
- Added `how-to/07-use-with-cursor.md` and `examples/cursor-mcp.json` — Cursor uses the `mcpServers` key (like Claude Desktop, not VS Code's `servers`), and the guide documents the Windows `dnx` SSL/timeout workaround (full `dotnet.exe` path + cached DLL).
- Fixed a factual error in `how-to/README.md`: the tool list omitted `get_document_info`, and it incorrectly claimed `get_supported_formats` requires a license to `Save()`. Both `get_supported_formats` and `get_document_info` are read-only and run fully in evaluation mode; only `convert` writes output (watermarked in eval).

## Why
Keeps the integration suite pinned to the current published server (26.7.0, which ships engine 26.6.0 and the shared `ToolError` error contract) and rounds out the client-setup guides with Cursor. `ToolDiscoveryTests` continues to assert exactly 3 tools — unchanged tool surface.

## Migration / impact
- Test-only repo — no runtime artifact. The suite exercises `GroupDocs.Conversion.Mcp@26.7.0` on nuget.org once that version is published; until then, override with `-p:McpPackageVersion=<older>` to run against a prior release.
